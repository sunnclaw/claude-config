# Sequential Thinking MCP 自动调用规则

当遇到以下情况时，自动调用 `mcp__sequential-thinking__sequentialthinking` 进行多步推理。

## 自动触发条件

### 1. 复杂决策类
- 架构设计、技术选型
- 多方案对比分析
- 权衡取舍类问题（"应该用 A 还是 B"）

### 2. 多步骤问题类
- 问题需要分解成多个子问题
- 需要 3 步以上的推理链
- 计划制定（项目计划、实现计划）

### 3. 调试分析类
- Bug 根因不明显
- 需要多角度排查
- 修复方案不明确

### 4. 探索研究类
- 开放性问题，没有唯一答案
- 需要自我修正思路
- 分支思考有助于全面分析

## 调用方式

开始时使用 `thoughtNumber: 1`，根据问题复杂度设置 `totalThoughts`（通常 3-7 步）。

设置 `nextThoughtNeeded: true`，直到得出最终结论后才设为 `false`。

当推理过程中发现之前思考有误，使用 `isRevision: true` 进行修正。

## 不触发的情况

- 简单直接的问题（查天气、问时间）
- 已有明确答案的查询
- 单步即可完成的任务
- 用户明确要求快速回答时
